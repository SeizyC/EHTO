# EHTO 프로젝트 : Everyone Has Their Own World (도메인:ehto.world)

## 1. 개발 목표

PRD V1.0의 핵심 목표는 단순 AI 채팅 서비스가 아니라, 사용자가 자신만의 작은 AI 사회 안에 머무르는 경험을 만드는 것이다. MVP는 한국어 기반 개인 AI 세계 1개를 중심으로, 다음 감각을 우선 구현한다.

- 사용자가 없어도 살아있는 듯한 월드 피드
- AI 멤버 간 관계, 분위기, 사건의 누적 변화
- 채팅창이 아닌 Social Ambient Feed
- World Identity Drift가 화면과 대화에 반영되는 구조
- 반복 피로도를 줄이는 이벤트, 리듬, 화제 전환

## 2. MVP 범위

### 포함

- 사용자별 개인 World 자동 생성
- 기본 AI 멤버 12명 생성 및 5명 내외 활성화 (AI를 내세우거나 강조하지 말고 ui상에선 AI란 verb 제외)
- World Feed: 대화, 프레즌스, 이벤트, 미디어, 월드 변화 로그
- World State Manager
- Social Director Engine
- Relationship Mutation Engine
- Narrative Event Engine
- Ambient Rhythm System
- User Signal Tracker
- Social Signature Builder
- Repetition Detection Engine 기본형
- World Identity 화면
- 한국어 대화 생성 파이프라인

### 제외 또는 후순위

- 실제 타 유저 월드 탐험
- 유료 티켓 결제
- 광고/스폰서 월드
- 고도화된 미디어 검증 자동화
- 대규모 샤딩 및 멀티 리전 확장

## 6. Agent 및 Engine 책임

### Main World Brain

월드 전체 상태를 읽고, 어떤 엔진을 호출할지 결정한다. 직접 모든 메시지를 생성하지 않고, 사회적 리듬과 방향성을 조율한다.

### Social Director Engine

- 다음 발화자 선택
- 침묵 유지 여부 결정
- 유저 호출 여부 결정
- AI끼리 대화할지 판단
- 화제 전환 필요성 판단

### Conversation Timing Agent

- typing delay
- delayed reaction
- interrupted typing
- sudden burst
- silent period

### Relationship Mutation Engine

- 메시지, 반응, 이벤트 후 관계 수치 변경
- 급격한 변화를 제한하고 누적 상호작용을 우선 반영

### Narrative Event Engine

- 작은 사회적 사건 생성
- 사건이 대화, 관계, 월드 분위기에 흔적을 남기도록 처리

### Repetition Detection Engine

- 동일 표현, 반응, 감정 패턴, 링크 타입 반복 감지
- 반복 위험이 높으면 topic injection, 이벤트 생성, 활동 멤버 변경을 제안

### Context Memory Agent

- 최근 대화 요약
- 내부 밈
- 관계 히스토리
- 월드 문화
- 장기 기억 압축

## 7. Tick 설계

### Fast Tick

- 주기: 2~10초
- 대상: 활성 세션
- 기능: typing, reaction, 짧은 답변, presence 변화

### Slow Tick

- 주기: 5~30분
- 대상: 활성/비활성 월드
- 기능: 관계 변화, 분위기 변화, 소규모 이벤트 생성

### Deep Tick

- 주기: 하루 1~3회
- 대상: 모든 월드
- 기능: World Identity Drift, memory compression, ritual 형성

## 9. 테스트 전략

- 도메인 엔진은 unit test 우선
- World State mutation은 snapshot test
- Agent decision은 fixture 기반 golden test
- Realtime feed는 integration test
- UI는 핵심 viewport별 Playwright 확인
- 모든 페이지는 모바일 360px, 390px viewport에서 레이아웃과 네비게이션 확인
- 반복 방지, safety, memory compression은 회귀 테스트 케이스 축적

## 11. 성공 기준

- 첫 진입 10초 안에 사용자가 “방이 이미 살아있다”는 느낌을 받는다.
- 사용자가 말하지 않아도 피드가 자연스럽게 흐른다.
- 같은 입력을 반복해도 월드 반응이 기계적으로 반복되지 않는다.
- 월드 분위기, 관계, 이벤트가 누적되어 World Identity 화면에 반영된다.
- UI가 카카오톡 복제처럼 보이지 않고, 사회적 공기와 관찰 경험을 제공한다.
- 모든 주요 페이지가 모바일에서 깨짐 없이 읽히고 조작된다.
